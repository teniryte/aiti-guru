# Тестовое задание Aiti Guru

При выполнении тестового задания приоритет был отдан архитектуре и масштабируемости.

Значительная часть задач, связанных с версткой и типовым функционалом компонентов для ускорения разработки демо-функцинала была реализована с помощью ИИ-инструментов (Cursor/Figma MCP/Codex). Примеры запросов — в документе [AI Requests](docs/ai-requests.md). Весь ИИ-код был использован в качестве базового, был в обязательном порядке отревьюен и при необходимости доработан.

В стилях лейаута страницы продуктов (в первую очередь, в шапке) позволил себе немного отойти от макета, чтобы диайн был более более консистентным.

В шапке страницы товаров добавлена информация о текущем пользователе и кнопка выхода.

## Установка, настройка и запуск

### Установка зависимостей

```sh
bun install
```

### Переменные среды

Для запуска дев-сервера можно скопировать файл с примерами переменных среды:

```sh
cp .env.example .env
```

### Запуск дев-сервера

```sh
bun run dev
```

Запуск дев-сервера происходит на порту `5050` по-умолчанию: [http://127.0.0.1:5050](http://127.0.0.1:5050).

## Технологический стек проекта

- **bun**: Эффективная среда выполнения и менеджер зависимостей.
- **TypeScript (strict)** / **React 19+**
- **Vite** — быстрый dev server и быстрая сборка, webpack слишком тяжелый и шумный для такого проекта.
- **FSD** — архитектура проекта. Хорошо раскладывает проект на слои, разделяет ответственность между UI, бизнес-логикой, API и инфраструктурой.
- **TanStack Router** — маршрутизация и URL-state. Типобезопасный, хорошо работает с работает с URL search params.
- **TanStack Query** — server state менеджер. Используется для получения, кэширования и рефетчинга серверных данных (продукты).
- **Axios** — HTTP-клиент.
- **Zustand** — client/UI state менеджер. Используется для д хранения состояния модалок, toast-очереди. Проще redux, целесообразнее для такого проекта.
- **React Hook Form** — формы. Эффективнее, чем Formik.
- **Zod** — валидация и схемы. Типобезопасный, интегрируется с RHF.
- **Radix UI** — headless UI-примитивы. Используется как ядро ui-компонентов со сложной механикой (диалоги, тосты, слайдер, прогрессбар). MUI и Ant тянут тяжелый визуальный слой, навязывают дизайн. Слишком много перезаписывать.
- **SCSS Modules** — стилизация компонентов. Дает удобную компактную организацию классов и свойств.
- **@lodash** — js-утилиты.
- **svgr (vite-plugin-svgr)** — загрузка svg-иконок из файлов в виде компонентов.
- **ESLint + Prettier** — качество и единообразие кода.

## Иконки

Файлы svg-иконок располагаются в директории src/shared/assets/icon.

Файл index.tsx определяет универсальный компонент `Icon`, который обертывает конкретную иконку (для консистентности).

## Стили

Файлы темы стилей располагаются в директории `src/app/styles`:

- `tokens.scss`: сырые значения;
- `theme-light.scss`: семантические переменные;
- `base.scss`: глобальные стили;
- Компонентные стили: в модулях `[component].module.scss` рядом с tsx-файлом компонента.

## UI-kit

Универсальные UI-компоненты расположены в директории `src/shared/ui`. Простые компоненты — кастомные, в сложных (диалоги, тосты и т.д.) используются примитивы radix-ui.

# Провайдеры

Сборка базовой инфраструктуры — в директории `src/app/providers`.

## Роуты и страницы

В проекте используется файловый роутинг TanStack Router (директория `src/routes`).

Слой страниц (оркестрация) — в директории `src/pages`.

## Остальные слои

- `widgets`: композитные блоки интерфейса;
- `features`: пользовательские сценарии;
- `entities`: бизнес сущности (ui, типы, хранилища, api).
